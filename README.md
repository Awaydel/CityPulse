# EcoSense Analytics

## О проекте
**Тема:** Агрегатор экологических данных с прогнозированием PM2.5.
**Цель:** Создать работающий прототип системы, которая собирает данные, хранит их в БД и отображает на веб-дашборде.

## Что мы сделали

1.  **Backend & ETL**: Настроили сбор данных с Open-Meteo (погода + загрязнение). Скрипт `etl.py` чистит данные и складывает их в PostgreSQL.
2.  **Machine Learning**: Добавили простую модель (Linear Regression), которая пытается угадать уровень пыли (PM2.5) по температуре и ветру.
3.  **Frontend**: Сделали сайт на React, где можно выбрать город и посмотреть графики.
4.  **QA & Аналитика**: Провели тестирование API и данных.

## Инструкция по запуску

Чтобы проверить работу проекта на локальной машине:

1.  **База данных**:
    *   Убедитесь, что PostgreSQL работает.
    *   Создайте базу `ecosense`.
    *   Запустите скрипт `ecosense.sql` для создания таблиц.

2.  **Бэкенд**:
    ```bash
    # В папке проекта
    python -m venv venv
    # Активация: venv\Scripts\activate (Windows) или source venv/bin/activate (Linux)
    pip install -r requirements.txt
    
    # 1. Загрузить данные
    python services/etl.py
    
    # 2. Запустить сервер
    python server.py
    ```

3.  **Фронтенд**:
    ```bash
    npm install
    npm run dev
    ```
    Откройте браузер по адресу `http://localhost:5173`.
