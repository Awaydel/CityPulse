# ПРОЕКТНАЯ ЗАДАЧА 5: АГРЕГАТОР "КАЧЕСТВО ВОЗДУХА + ПОГОДА"
=========================================================

Проект реализует автоматизированный ETL-конвейер для сбора, унификации и контроля качества данных (Data Quality) о загрязнении воздуха (OpenAQ) и метеорологических факторах (Open-Meteo) по 5 крупным городам.

**Ключевая цель:** Построение воспроизводимой аналитической витрины (Star Schema) и дашборда для исследования *корреляций* между погодой и уровнем PM2.5/PM10.

## 2. Структура Репозитория
-------------------------

Структура проекта соответствует принципу Doc-as-code и обеспечивает легкую навигацию:

| Директория | Содержание файла |
| :--- | :--- |
| `/airflow` | Код DAG'ов, хуки и конфигурации Airflow. |
| `/sql` | Скрипты создания витрины, ETL-запросы. |
| `/docs` | **Вся проектная документация (Критерии приёмки).** |
| `/docs/ADR` | Архитектурные и ключевые аналитические решения. |
| `/scripts` | Вспомогательные скрипты для настройки Dev-Env. |
| `/notebooks` | Черновые ноутбуки для EDA и статистического анализа. |
