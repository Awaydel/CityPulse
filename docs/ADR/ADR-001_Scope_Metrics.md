# ADR-001: Выбор Скоупа и Унификация Метрик
==========================================

**Статус:** Принято

**Дата Принятия:** 2025-11-15 

---

## 1. Контекст Решения
----------------------
Проект требует сбора и сопоставления данных о загрязнении и погоде из двух внешних API. Для демонстрации корреляции и выявления рисков, необходим скоуп, представляющий различные климатические и регуляторные зоны.

## 2. Решение 1: Выбор Городов
-----------------------------

Выбрано 5 городов для обеспечения наглядности и разнообразия факторов (климат, широта, экстремальность загрязнения):

| Город | Страна | Часовой Пояс (зимнее) | Ключевой Фактор |
| :--- | :--- | :--- | :--- |
| Москва | Россия | UTC+3 | Высокая сезонность, континентальный климат. |
| Мехико | Мексика | UTC-6 | Высокогорье, проблемы смога. |
| Лондон | Великобритания | UTC+0 | Умеренный климат, развитый мониторинг. |
| Пекин | Китай | UTC+8 | Экстремальные пики PM2.5/PM10. |
| Сидней | Австралия | UTC+10 | Южное полушарие, влияние лесных пожаров (риск). |

## 3. Решение 2: Семантическая Унификация Метрик
---------------------------------------------

Все данные, загружаемые в аналитическую витрину, будут преобразованы к следующим стандартам для обеспечения согласованности и качества (QA requirement):

| Показатель | Унифицированная Единица | Обоснование |
| :--- | :--- | :--- |
| Температура | Градусы Цельсия (°C) | Стандарт метеоданных. |
| Скорость ветра | Метры в секунду (м/с) | Стандарт SI. |
| Загрязнители (PM2.5, PM10, O3 и т.д.) | Микрограммы на кубический метр (μg/m³) | Стандарт OpenAQ и ВОЗ. |
