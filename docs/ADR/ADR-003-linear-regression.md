# ADR-003: Linear Regression для прогнозирования PM2.5

## Статус

✅ Принято

## Контекст

ТЗ требует реализовать ML-модель для прогнозирования PM2.5. Нужно выбрать алгоритм и подход к обучению.

## Решение

Используем **Linear Regression** из scikit-learn с признаками: температура, влажность, скорость ветра.

## Обоснование

| Алгоритм | Плюсы | Минусы |
|----------|-------|--------|
| Linear Regression | Простой, интерпретируемый, быстрый | Линейные зависимости |
| Random Forest | Нелинейные зависимости | Сложнее, дольше обучение |
| Gradient Boosting | Высокая точность | Требует tuning, больше данных |
| Neural Network | Гибкость | Overkill для задачи |

**Выбор:** Linear Regression — достаточно для MVP и демонстрации концепции.

## Реализация

```python
# services/etl.py
from sklearn.linear_model import LinearRegression

X = df[['temperature', 'wind_speed', 'humidity']]
y = df['pm25']

model = LinearRegression()
model.fit(X, y)

df['predicted_pm25'] = model.predict(X)
```

## Последствия

### Положительные
- Быстрое обучение (< 1 сек)
- Понятные коэффициенты
- Не требует GPU

### Отрицательные
- Может недооценивать экстремальные значения
- Не учитывает временные зависимости (лаги)

## Развитие

Для улучшения можно:
1. Добавить лаговые признаки (PM2.5 за предыдущий час)
2. Перейти на Gradient Boosting (XGBoost/LightGBM)
3. Добавить back-testing для оценки качества
