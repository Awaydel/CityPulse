# DASHBOARD_SPECS.md: Техническое Задание на Визуализацию
========================================================

## 1. Общая Цель
------------------
Создать аналитический дашборд, который выполняет три ключевые задачи: 1) Визуализация трендов (Спрос/Нагрузка), 2) Оценка корреляций (Матрица риска), 3) Контроль качества данных (Отчет).

## 2. Структура Дашборда и Требования к Страницам

### Страница 1: Спрос / Нагрузка (Тренды)

*   **Ключевой элемент:** **Тепловая карта (Heatmap)** PM2.5 по часам и дням недели для каждого города.
    *   *Источник данных:* `fact_air_quality` и `dim_time`.
    *   *Назначение:* Визуализировать, в какое время суток и дни недели наблюдаются максимальные загрязнения (Требование 7).
*   **График 1:** Временной тренд PM2.5/PM10 по месяцам (линейный график).
    *   *Интерактивность:* Выбор города.

### Страница 2: Качество и Корреляции (Анализ Влияния)

*   **Ключевой элемент:** **Матрица Корреляций** (Correlation Matrix).
    *   *Содержание:* Коэффициенты корреляции Пирсона между PM2.5 и ключевыми метеопоказателями (Температура, Скорость ветра, Осадки) для каждого города.
    *   *Назначение:* Прямая демонстрация минимальных метрик проекта.
*   **Виджет 1:** **«Карта Риска»** (Требование 7).
    *   *Описание:* Сводная таблица или светофор, показывающий количество дней, когда PM2.5 превышал лимиты ВОЗ (из `dim_metric`) за последний месяц, по каждому городу.

### Страница 3: Отчет о Качестве Данных (QA Review)

*   **График 1:** **Доля Пропусков (Data Completeness)**.
    *   *Описание:* Столбчатая диаграмма, показывающая долю пропущенных данных (показатель `data_completeness` или доля NULL/Not Found) по каждому городу/метрике.
    *   *Назначение:* Демонстрация результата работы QA.
*   **Виджет 1:** **Агрегированный Отчет DQ-2 (Красный Флаг)**.
    *   *Описание:* Текстовый виджет, отображающий последнее зафиксированное значение "Красного Флага" (доля нарушений DQ-2) и дату его срабатывания.
