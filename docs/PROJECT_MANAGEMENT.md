## 6. Процессы и методология

### 6.1 Методология разработки
**Agile Scrum** с 2-недельными спринтами

### 6.2 Workflow Git
1. Создание feature branch от `main`
2. Разработка и локальное тестирование
3. Pull Request с описанием изменений
4. CI/CD проверки (тесты + coverage)
5. Merge в `main`

### 6.3 Definition of Done (DoD)

Задача считается завершённой, если:
- [x] Код написан и соответствует стандартам
- [x] Unit-тесты написаны (покрытие ≥ 70%)
- [x] Интеграционные тесты пройдены
- [x] CI/CD pipeline успешно выполнен
- [x] Документация обновлена

## 7. Управление изменениями

### 7.1 Процесс Change Request

1. **Инициация:** Запрос на изменение через PM или команду
2. **Оценка влияния:** Анализ влияния на сроки, ресурсы, архитектуру
3. **Решение:** Принятие/отклонение с обоснованием
4. **Планирование:** Включение в бэклог при принятии
5. **Реализация:** Стандартный workflow разработки
6. **Фиксация:** Обновление документации и ADR (Architecture Decision Record)

### 7.2 ADR (Архитектурные решения)

Все значимые архитектурные решения документируются в `docs/ADR/`.

**Пример:** [`ADR-001: Выбор городов и метрик`](file:///c:/Users/Андрей/air_quality_aggregator/docs/ADR/ADR-001_Scope_Metrics.md)
